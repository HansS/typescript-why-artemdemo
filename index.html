TypeScript

Зачем, почему и что мне с того

Что это?

  • Суперсет JavaScript'а
  • Создан для упрощения разработки и поддержки больших проектов
  • На выходе компилируется в JavaScript (куда же он денется)

Зачем это нужно? (коротенечко)

  • Используем синтаксис и фичи ES2015 (+ часть ES2016)
  • Работа с типами упрощает отлов багов до того, как они попадут в консоль браузера
  • Типы и интерфейсы упрощают работу IDE - подсказок больше ^_^

Побочный эффект введения - фронтендеры будут жаловаться на "надоевшие" ошибки типирования.

Generics


interface IPromise<T> {
    then<TResult>(
        successCallback: (promiseValue: T) => any,
        errorCallback?: (reason: any) => any,
        notifyCallback?: (state: any) => any
    ): IPromise<TResult>;

    catch<TResult>(
        onRejected: (reason: any) => any;
    ): IPromise<TResult>;

    finally(
        finallyCallback: () => any
    ): IPromise<T>;
}
            

public getProjects(): angular.IPromise<IProject[]> {
    let deferred: angular.IDeferred<IProject[]> = this.$q.defer();

    this.$q.all([
        this.projectsLoadingPromise,
        this.tasksService.getTasksLoadingPromise()
    ]).then(
        (projects) => deferred.resolve(projects),
        (error) => deferred.reject(error)
    );

    return deferred.promise;
}

Typescript vs Coffee

  • Валидный JS, валиден и для TS
  • TS это тот же JS, только с типированием

Не зная TS, можно понять о чем речь смотря на код. В случае c CS это не так очевидно.

Typescript vs Babel

  • Оба поддерживают почти весь синтаксис ES6 (Babel поддерживает больше)
  • Оба поддерживают модули через сторонние библиотеки: common, amd, system
  • Оба начинают внедрять ES7: Decorators

У обоих есть будущее в поддержке следующих стандартов ECMAScript'а

Typescript ❤

Однако Typescript в более выгодном положении

  • Microsoft
  • Typescript активно сотрудничает с разработчиками JS библиотек: Angular, Ember

Typescript для больших проектов

  • Ловим ошибки на стадии компиляции
  • Компилятор заставит писать более качественный код
  • Весь код валидируется при компиляции
  • Как следствие - при рефакторинге проще ловятся ошибки
  • Для все основных библиотек уже есть файлы с описанием типов
